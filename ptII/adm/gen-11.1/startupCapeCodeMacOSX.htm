Starting CapeCode

(The contents of this file are displayed during installation and also appear as $PTII/startupCapeCodeMacOSX.htm)

CapeCode.app

Under Mac OS X, CapeCode includes $PTII/bin/CapeCode.app, where $PTII is the location where CapeCode was installed. Users that want to run CapeCode under Mac OS X typically double click on CapeCode.app.

$PTII/bin/capecode

If you would like to extend or modify CapeCode, then set the $PTII environment variable to point to the location of your CapeCode tree.

This is typically done by editing ~/.bashrc and adding: 

      export PTII=/Applications/capecode0.3.devel

where /Application/capecode0.3.devel is the location where CapeCode was installed.

After setting $PTII, to rebuild use: 

      cd $PTII
      ./configure
      $PTII/bin/ant
      $PTII/bin/capecode

See also Ptolemy II/CapeCode.

Details

Ptolemy II and CapeCode consist of multiple applications it it is expected that the user will browse the distribution, so we don't ship a single .app directory. In addition, for support reasons, we want the distributions to be fairly uniform between the different platforms.

In addition, we hope that most users will extend Ptolemy II/CapeCode and rebuild it. Rebuilding Ptolemy II and CapeCode creates the Mac .app files in $PTII/bin. If we had a top level .app file for CapeCode, this would clutter the top level directory. In addition, for CapeCode, we would end up with $PTII/Vergil.app, $PTII/Ptiny.app, $PTII/CapeCode.app and possibly other .app files.

Other multi-platform applications include a bin directory on the Mac that includes the .app directories.
